up.util.cssAnimate

Syntax

up.util.cssAnimate

(
  • elementOrSelector
  • lastFrame
  • [opts.duration=300]
  • [opts.delay=0]
  • [opts.easing='ease'] The timing function that controls the animation's acceleration. See [W3C documentation]
)

Summary

Animates the given element's CSS properties using CSS transitions.

Parameters:

  • elementOrSelector Element | JQuery | String

    The element to animate.

  • lastFrame Object

    The CSS properties that should be transitioned to.

  • [opts.duration=300] Number optional

    The duration of the animation, in milliseconds.

  • [opts.delay=0] Number optional

    The delay before the animation starts, in milliseconds.

  • [opts.easing='ease'] The timing function that controls the animation's acceleration. See [W3C documentation] String

    (http://www.w3.org/TR/css3-transitions/#transition-timing-function) for a list of pre-defined timing functions.

Returns:

A promise for the animation's end.

up.util.option

Defined in lib/assets/javascripts/up/util.js.coffee:195

Syntax

up.util.option

(
  • args...
)

Summary

Returns the first argument that is considered present. If an argument is a function, it is called and the value is checked for presence.

This function is useful when you have multiple option sources and the value can be boolean. In that case you cannot change the sources with a || operator (since that doesn't short-circuit at false).

Parameters:

  • args... Array
